---
description: Як оновити код на GitHub (Git Workflow)
---

Цей воркфлоу описує кроки для безпечного оновлення репозиторію після завершення завдань.

// turbo
1. Перевірити наявність змінених файлів:
   `git status`

2. Додати всі зміни до індексу:
   `git add .`

3. Створити коміт з **реальним описом**. (Я завжди буду надавати вам готову команду з актуальним текстом):
   `git commit -m "Ваш реальний опис роботи"`

4. Відправити зміни в основну гілку (або поточну):
   `git push origin main`

> [!TIP]
> Завжди перевіряйте статус перед пушем, щоб уникнути відправки непотрібних тимчасових файлів.
